# 📘 Учебник по архитектуре AI Character Chat (Node.js + TypeScript)

Добро пожаловать в документацию проекта. Этот файл — твой путеводитель по миру современного бэкенда на Node.js, адаптированный под опыт разработки на Laravel.

---

## 1. Архитектура: Коротко о главном

Мы разделили проект на слои (Layers), чтобы код не был "макаронами":
- **`config/`**: Константы и настройки из `.env`.
- **`database/`**: Работа с SQLite (паттерн Repository).
- **`routes/`**: Входные точки API (Контроллеры).
- **`services/`**: Бизнес-логика (AI, работа с картинками).
- **`types/`**: Твои интерфейсы (TypeScript контракты).

---

## 2. Сессии: Как в Laravel/PHP

Мы отказались от JWT в пользу более привычных тебе сессий.

### Как это устроено:
1.  **Библиотеки**: Мы используем `@fastify/cookie` и `@fastify/session`.
2.  **Login**: При успешном входе мы пишем: `request.session.user = { id: 1 ... }`. Это полный аналог `$_SESSION` в PHP.
3.  **Cookies**: Сервер сам отправляет браузеру куку `session_id`. Твой браузер будет слать её обратно при **каждом** клике. Тебе не нужно вручную прописывать заголовки в JavaScript (как при JWT).
4.  **Middleware (authenticate)**: Мы создали декоратор `server.authenticate`. Он проверяет: "Есть ли запись о пользователе в сессии?". Если нет — 401.

---

## 3. Слой Данных (Repository Pattern)

В Laravel ты привык к Eloquent. Здесь у нас есть `dbRepo`:
- Вместо `User::where('email', $email)->first()`, ты пишешь `await dbRepo.getUserByEmail(email)`.
- Это позволяет легко тестировать код и менять БД, не задевая логику роутов.

---

## 4. AI Service (Мозги системы)

Это место, где происходит магия:
- **Streaming**: Мы отдаем ответ нейронки по буквам (`Server-Sent Events`). Пользователь не ждет 30 секунд полного ответа.
- **Image Processing**: Используем `sharp` для сжатия фото. Нейронки не любят огромные файлы.
- **Context Summarization**: Если история чата слишком длинная, сервис просит ИИ кратко пересказать начало (Summarize) и удаляет старые сообщения из базы.

---

## 5. TypeScript для PHP-шника

- **Interface**: Это твой новый DocBlock. Вместо `@property string $name`, ты пишешь `interface Character { name: string }`.
- **Await**: В Node.js почти всё — асинхронное. Если функция обращается к диску или сети, перед ней ВСЕГДА будет `await`.
- **node_modules**: Твой новый `vendor`.

---

## 6. Настройка параметров ИИ (Крутилки и рычаги)

Если ИИ начал бредить или отвечать слишком коротко, тебе нужно знать, где подкрутить настройки. У нас три уровня контроля:

### А. Глобальные настройки (`.env`)
Это база. Без них магии не будет:
- **`API_URL`**: Куда стучимся (OpenRouter, OpenAI или твой локальный прокси).
- **`AI_DEFAULT_MODEL`**: Какую модель юзаем по умолчанию (например, `x-ai/grok-4.1-fast`).
- **`JWT_SECRET`**: Секретный ключ для сессий.

### Б. Лимиты контекста и логика (`src/backend/config/config.ts`)
Тут живут системные "тормоза" и продвинутые настройки ИИ (меняются прямо в коде):
- **`aiTemperature`** (дефолт: 0.7): Глобальная температура.
- **`aiTopP`** (дефолт: 0.9): Параметр Top-P.
- **`aiFrequencyPenalty`** (дефолт: 0.5): Штраф за повторение слов.
- **`aiPresencePenalty`** (дефолт: 0.5): Штраф за повторение тем.
- **`aiSafePrompt`** (дефолт: true): Безопасный промпт.
- **`aiProvider`**: Настройки провайдера. По умолчанию `sort: 'throughput'`.
- **`aiReasoning`**: Настройки рассуждений. По умолчанию `effort: 'none', exclude: true`.
- **`maxHistoryMessages` (дефолт: 20)**: Сколько сообщений отправляем ИИ.

### В. Тюнинг персонажа (База данных / UI)
У каждого персонажа в таблице `characters` есть свои "характерные" настройки. Если они не заданы (null), система берет значения из `config.ts` (Fallback):
- **`temperature`** (дефолт: 0.7): Уровень "творчества".
- **`max_tokens`** (дефолт: 200): Ограничитель длины ответа.
- **`system_prompt`**: Самая важная часть. Тут ты прописываешь: "Ты — дерзкий хакер" или "Ты — милая горничная".
- **`scenario`**: Описание ситуации (локация, текущее событие).

### Г. Скрытые параметры (Hardcoded)
Кое-что зашито в глубине кода, чтобы всё работало "из коробки":
- **Обработка картинок**: Всё сжимается до **1024x1024** с качеством **80%** (в `ai.service.ts`). Нейронкам этого за глаза, а твой трафик страдать не будет.
- **Интервал печати (Frontend)**: В `app.js` задержка вывода символов составляет **15мс** (или **2мс**, если очередь сообщений слишком большая). Чтобы текст "бежал" плавно, а не вываливался куском.
- **Markdown**: Включен перенос строк (`breaks: true`) и подсветка кода через `highlight.js`.

---

### Как пользоваться проектом:
1. Настрой `.env` (API_KEY, API_URL, JWT_SECRET).
2. Запусти `npm run seed` для создания своего админа и первого персонажа.
3. Запусти `npm run dev` и начни творить.
